# HireFlow AI 招聘提效工作台

HireFlow 是一个面向 HR 招聘运营的 PC 端 AI 应用 Demo，用最小成本展示「企业微信 / 企业微信群 / 腾讯在线文档」招聘场景中的自动化落地能力：AI 自动抽取候选人信息、实时更新招聘台账、同步状态可视化、异常风险提醒。

## 在线/本地运行

这是纯静态 Demo，无需安装依赖。

1. 克隆或下载本仓库。
2. 直接用浏览器打开 `index.html`。
3. 点击「导入今日群聊」查看 AI 招聘台账自动生成效果。

## 解决的问题

现有招聘流程中，HR 需要在企业微信、群聊和腾讯在线文档之间反复切换，手动记录候选人、岗位、阶段、面试时间、负责人和跟进状态。常见问题包括：

- 人工录入耗时，重复劳动多。
- 群聊信息碎片化，容易漏记候选人或面试安排。
- 腾讯文档更新不及时，招聘负责人无法实时看到进展。
- 缺字段、重复建档、超时未跟进等异常依赖人工发现。

## Demo 能力

- 企业微信群聊消息导入模拟。
- AI 自动抽取候选人结构化字段。
- 招聘漏斗和核心指标自动刷新。
- 腾讯在线文档式台账实时更新。
- 缺失字段、重复候选人、待确认数据自动提醒。
- 支持按岗位和负责人筛选。

## 最小成本落地方案

### 第一阶段：3 日内可交付 Demo

- 采用 `HTML + CSS + JavaScript` 静态页面。
- AI 抽取能力用前端模拟，但字段结构按真实大模型 JSON 输出设计。
- 可部署到 GitHub Pages、Vercel 或 Netlify，形成在线预览链接。

### 第二阶段：低成本 MVP

- 消息入口：优先使用企业微信群机器人、固定格式转发或 HR 手动粘贴群聊文本，降低企业微信会话存档审批成本。
- AI 抽取：调用低成本大模型 API，将聊天记录解析为结构化 JSON。
- 数据存储：使用腾讯在线文档、腾讯表格或轻量数据库保存招聘台账。
- 同步机制：定时任务或 webhook 更新台账和看板。
- 人工确认：缺失字段、重复候选人、冲突记录进入 HR 确认队列。

### 第三阶段：自动化增强

- 接入企业微信会话存档或审批后的消息回调。
- 与 ATS、腾讯文档 API、日历系统联动。
- 增加候选人意向评分、简历匹配度、面试超时提醒。
- 建立岗位、负责人、渠道、招聘周期等经营分析看板。

## AI 模型应用方式

真实落地时，模型输入为企业微信消息、群聊文本或腾讯文档新增备注，输出统一 JSON：

```json
{
  "name": "张晨",
  "role": "前端工程师",
  "source": "Boss 直聘",
  "stage": "待一面",
  "schedule": "6 月 18 日 14:00",
  "owner": "Alice",
  "riskTags": ["重复候选人", "信息已补充"],
  "confidence": 0.93
}
```

推荐策略：

- 先用规则校验关键字段，如姓名、岗位、阶段、负责人。
- 再用大模型完成自然语言理解、字段补全、标签生成。
- 对低置信度、缺字段、疑似重复记录进入人工确认。
- 对候选人手机号、微信号等敏感信息做脱敏或权限控制。

## 成本控制

- Demo 阶段零后端、零数据库、零模型调用成本。
- MVP 阶段优先复用企业微信、腾讯在线文档和云函数。
- 模型调用只处理新增消息和变更内容，避免重复消耗 token。
- 高频字段用规则抽取，复杂语义再调用大模型。
